ИЗМЕНЕНИЕ #1: СИСТЕМА ДВИЖЕНИЯ ТЕЛА 
===============================================================================

СУТЬ: Заменили прямую привязку параметров к движению рта на систему состояний 
(idle/talking) с многослойным шумом, импульсами и физической инерцией.

ФАЙЛЫ:
======
• bodyMovement.js - новый файл с системой движений
• live2d.js - интеграция, отключение старой привязки, функция notifyMouthActivity
• window.html - UI секция "Body Movement System"
• ui.js - 5 обработчиков событий для настроек
• index.js - настройки по умолчанию и инициализация событий

АЛГОРИТМ:
=========
1. Отслеживание: playTalk() → notifyMouthActivity(true/false)
2. Состояния: idle (500мс+ без движения рта) / talking
3. Шумы: 3 слоя (медленный/средний/быстрый) + импульсы + корреляция
4. Физика: пружина-демпфер для инерции
5. Применение: преобразование в диапазон параметра модели

НАСТРОЙКИ:
==========
• bodyMovementEnabled: bool, по умолчанию true
• bodyMovementIdleIntensity: 0.0-1.0, по умолчанию 0.3  
• bodyMovementTalkingIntensity: 0.0-1.0, по умолчанию 0.6
• bodyMovementImpulseChance: 0-10%, по умолчанию 2%
• bodyMovementSmoothness: 0.5-0.99, по умолчанию 0.85
• bodyMovementImpulseInertia: 0.80-0.98, по умолчанию 0.92

КЛЮЧЕВЫЕ ФУНКЦИИ:
=================
• startBodyMovement() - запуск системы для персонажа
• stopBodyMovement() - остановка системы
• restartBodyMovement() - перезапуск
• notifyMouthActivity() - уведомление о движении рта

ФАЙЛЫ:
======
• live2d.js - функции autoEyeMovement, autoMicrosaccades
• ui.js - обработчики событий для всех настроек анимаций  
• window.html - UI секция "Auto Animations" с детальными настройками
• index.js - настройки по умолчанию и инициализация событий

АЛГОРИТМЫ:
==========
1. ДВИЖЕНИЯ ГЛАЗ: система саккад-фиксаций с центральными/периферийными зонами
2. МИКРОСАККАДЫ: мелкие случайные движения поверх основных движений глаз
3. УПРАВЛЕНИЕ: startAutoAnimations/stopAutoAnimations/restartAutoAnimations

НАСТРОЙКИ:

• microsaccadesEnabled: bool, включение микросаккад, по умолчанию true
• microsaccadeAmplitude: 0.001-0.1, сила микросаккад, по умолчанию 0.02
• microsaccadeFrequency: 0.1-3.0, частота микросаккад, по умолчанию 1.0
• microsaccadeDuration: 5-50мс, длительность микросаккад, по умолчанию 15мс
• microsaccadeIntervalMin/Max: 100-1000/500-3000мс, интервалы между микросаккадами, по умолчанию 300/1500мс

ИЗМЕНЕНИЕ #6: TTS AUDIO BINDING СИСТЕМА (@4eckme) - ИСПРАВЛЕНО
===============================================================================

СУТЬ: Добавлена интеграция с внешней TTS системой для синхронизации движений рта 
с аудио воспроизведением вместо симуляции по тексту.

ФАЙЛЫ:
======
• live2d.js - модификация функции playTalk() с добавлением TTS bind

АЛГОРИТМ:
=========
1. Установка window.live2d_tts_bind = false в начале разговора
2. Цикл анимации рта работает по таймеру: text.length * mouth_time_per_character
3. ВНУТРИ цикла: персонаж закрывает рот и ждет сигнал от TTS (window.live2d_tts_bind === true)
4. Когда TTS дает сигнал - персонаж начинает двигать ртом
5. Цикл повторяется пока не истечет время или не произойдет прерывание
6. После завершения ВСЕГДА вызывается notifyMouthActivity(character, false)

КОД ИНТЕГРАЦИИ ДЛЯ ВНЕШНИХ TTS:
================================
```javascript
// TTS система должна периодически устанавливать флаг в true во время воспроизведения аудио:
window.live2d_tts_bind = true;

// Когда аудио заканчивается, TTS должна установить флаг в false:
window.live2d_tts_bind = false;
```

ОРИГИНАЛЬНАЯ ПРОБЛЕМА (@4eckme):
=================================
• Цикл имел условие `while ((Date.now() - startTime) < duration || true)`
• Часть `|| true` делала цикл БЕСКОНЕЧНЫМ для работы с TTS
• Код после цикла (включая notifyMouthActivity) НИКОГДА не выполнялся
• Body Movement System не получала сигнал о завершении разговора
• Персонаж застревал в режиме talking навсегда

ИСПРАВЛЕНИЕ - МОНИТОРИНГ РЕАЛЬНОГО ЗНАЧЕНИЯ РТА (Сентябрь 2025):
==================================================================
✓ ОСТАВИЛИ код @4eckme КАК ЕСТЬ (бесконечный цикл с `|| true`)
✓ ДОБАВИЛИ ОТДЕЛЬНУЮ систему мониторинга РЕАЛЬНОГО значения параметра рта
✓ Body Movement System теперь реагирует на РЕАЛЬНОЕ открытие рта, а не на флаги

МЕХАНИЗМ МОНИТОРИНГА:
=====================
1. При загрузке модели запускается startMouthMonitoring(character, model, model_path)
2. Запоминается БАЗОВОЕ значение рта в покое (baseline)
3. Фоновый цикл каждые 50мс проверяет РЕАЛЬНОЕ значение параметра рта через getParameterValueById
4. Вычисляется отклонение от baseline: deviation = |текущее значение - baseline|
5. Если deviation > 0.3 → рот активен → notifyMouthActivity(character, true) → talking
6. Если deviation <= 0.3 И прошло 500мс без активности → notifyMouthActivity(character, false) → idle
7. Уведомления отправляются ТОЛЬКО при изменении состояния (idle ↔ talking)

ПРЕИМУЩЕСТВА:
=============
• Независимость от логики playTalk и TTS binding
• Относительное отклонение вместо абсолютного порога (работает с любыми моделями)
• Реакция на реальное движение рта, а не на таймеры или флаги
• Персонаж начинает активно двигаться ТОЛЬКО когда рот РЕАЛЬНО двигается (звук голоса)
• Полная совместимость с любыми TTS системами
• Автоматическая работа без необходимости модификации playTalk
• Детальное логирование для отладки (baseline, текущее значение, deviation)

ПАРАМЕТРЫ:
==========
• MOUTH_MOVEMENT_THRESHOLD = 0.3 - порог отклонения для определения активности
• IDLE_TIMEOUT = 500мс - время без движения для перехода в idle
• CHECK_INTERVAL = 50мс - частота проверки значения рта
• baselineMouthValue - запоминается при старте мониторинга

ФУНКЦИИ:
========
• startMouthMonitoring(character, model, model_path) - запуск мониторинга
• stopMouthMonitoring(character) - остановка мониторинга
• mouthMonitoring[character] - флаг активности мониторинга

ЧТО НЕ ИЗМЕНИЛОСЬ:
==================
• Оригинальный код @4eckme полностью сохранён
• Логика TTS binding работает точно так же
• Бесконечный цикл с `|| true` остался
• Внешняя TTS система использует тот же API (window.live2d_tts_bind)

НАЗНАЧЕНИЕ:
===========
• Позволяет внешним системам TTS контролировать движения рта
• Синхронизация с реальным аудио вместо симуляции по тексту
• Используется сторонними разработчиками для улучшения реализма
• Теперь совместима с Body Movement System (корректные переходы idle/talking)
